## Key learnings

### FreqLeader (433 lines, 5 phases + failed)
Meet-and-Lead with nar hooks. State machine:
```
MEET → LEAD → HOLD → LEAD → SUSTAIN
             → FAILED (max_hold timeout, micro-step back)
             → SUSTAIN dropout: micro-step back to LEAD
```
- **Passthrough mode**: When control panel agent drives beat_frequency directly (conductor passthrough), the leader publishes freq_lead_data recommendations only — never overwrites beat_frequency. Keeps the two agents from fighting.
- **Fast-follower mode**: 3 holds-free consecutive steps → halve step_interval (floor 15s).
- **Nar integration**: `_narrate()` writes `agent_message` via patch_live(). In passthrough it skips writes.
- **Deactivation**: Restores beat_frequency to `eeg_iaf_hz` (fallback: current beat). Never snaps to hard-coded 10.
- **Alpha-overlap correction**: When `entrainment_confidence == "alpha_overlap"`, scales ASSR × 0.7 before phase decision.
- **FAILED**: Only `max_hold` timeout exists — pushes current_freq up by step_size, resets timer, retries. No terminal "give up" state. Holds loop blocks `_check_success` forever.
- **Sustain strategy**: `freq_lead_mode == "sustain"` skips MEET entirely.
